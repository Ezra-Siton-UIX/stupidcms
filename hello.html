## API Routes Overview ### כללי - כל collection במערכת נגיש דרך API. - הנתונים נשמרים כ־JSON בתיקיית `server/data//`. - אין כרגע query params לפילטור, אלא ראוטים נפרדים. ### קריאות זמינות לכל collection: #### רשימה כללית (public/private) - `GET /api/public/:site/:collection` - `GET /api/:site/:collection` (דורש הרשאה) #### רשימה לפי קטגוריה (public/private, רק לקולקשנים שתומכים בקטגוריות) - `GET /api/public/:site/:collection/by-category/:categoryId` - `GET /api/:site/:collection/by-category/:categoryId` (דורש הרשאה) #### דוגמה ל־posts: - `/api/public/site_bobby/posts` - `/api/public/site_bobby/posts/by-category/webflow` - `/api/site_bobby/posts` - `/api/site_bobby/posts/by-category/webflow` #### דוגמה ל־faq: - `/api/public/site_bobby/faq` - `/api/public/site_bobby/faq/by-category/loveable` - `/api/site_bobby/faq` - `/api/site_bobby/faq/by-category/loveable` #### הערות: - אין כרגע תמיכה ב־query params לפילטור, אלא רק ב־route יעודי. - כל route מחזיר מערך של אובייקטים מהקולקשן הרלוונטי. - routes פרטיים דורשים JWT. --- **המלצה:** להציג את הבלוק הזה (או דומה לו) תחת Developer debug בממשק הניהול, כדי שיהיה קל לראות מה זמין לכל collection. --- אם תרצה שאוסיף קוד שמייצר את זה אוטומטית בממשק הניהול, שלח לי את קובץ ה־admin שבו מרנדרים את ה־Developer debug.
